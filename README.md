# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Webpack

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

Важные файлы:
- src/pages/index.html — HTML-файл главной страницы
- src/types/index.ts — файл с типами
- src/index.ts — точка входа приложения
- src/scss/styles.scss — корневой файл стилей
- src/utils/constants.ts — файл с константами
- src/utils/utils.ts — файл с утилитами

## Установка и запуск

## Данные и типы данных, используемые в приложении

Товар

```
export interface IItem {
  id: string;
  description: string;
  image: string;
  title: string;
  category: string;
  price: number;
}
```

Пользователь

```
export interface IUser {
  payment: string;
  email: string;
  phone: string;
  address: string;
  total: number;
  items: IItem[]
}
```

Модель для хранения данных товаров

```
export interface IItemsData {
  total: number;
  items: IItem[];
  preview: string | null;
  getItem(itemId: string): IItem;
  updateItem(item: IItem, payload: Function | null): void;
}
```

Модель для хранения данных пользователя

```
export interface IUserData {
  getUserInfo(): TUserModal;
  setUserInfo(userData: IUser): void;
}
```

Тип для товара, отображаемого на главной странице

```
export type TItemPublic = Omit<IItem, 'id' | 'description'>;
```

Тип для товара, открывающегося в модальном окне просмотра (после клика по нему с главной страницы)

```
export type TItemModal = Omit<IItem, 'id'>;
```

Тип для товара, отображаемого в корзине

```
export type TItemBasket = Pick<IItem, 'title' | 'price'>;
```

Тип для данных пользователя, используемых в модальных окнах

```
export type TUserModal = Omit<IUser, 'items'>;
```

## Архитектура приложения

Код приложения разделен согласно парадигме MVP:
- слой представления, который отвечает за отображение данных на странице
- слой данных, который отвечает за хранение и изменение данных
- презентер, который отвечает за связь представленя и данных

### Базовый код

#### Класс Api
Содержит в себе базовую логику отправки запросов. В конструктор передается базовый адрес сервера и опциональный объект с загголовками запросов.
Методы:
- 'get' - выполняет GET запрос на переданный в параметрах end-point и возвращает промис с объектом, которым ответил сервер.
- 'post' - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на end-point, переданный как параметр при вызове метода. По умолчанию выполняется 'POST' запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.

#### Класс EventEmitter
Брокер событий позволяет отправлять события и подписываться на события, происходящие в системе. Класс используется в презентере для обработки событий и в слоях приложения для генерации событий.
Основные методы, реализуемые классом описаны интерфейсом 'IEvents':
- 'on' - подписка на событие
- 'emit' - инициализация события
- 'trigger' - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие

### Слой данных

#### Класс ItemsData
Класс отвечает за хранение и логику работы с данными товаров.\
Конструктор класса принимает инстант брокера событий.\
В полях класса хранятся следующие данные:
- _total: number - общее число товаров на странице
- _items: IItem[] - массив объектов товаров
- _preview: string | null - id товара, выбранного для просмотра в модальном окне
- events: IEvents - экземпляр класса `EventEmitter` для инициации событий при изменении данных.

Класс имеет набор методов для взаимодействия с данными:
- getItem(itemId: string): IItem - возвращает товар по его id
- updateItem(item: IItem, payload: Function | null): void - обновляет данные товара в массиве. Если передан колбэк, то выполняет его после обновления, если нет, то вызывает изменения массива.
- а так же геттеры для получения данных из полей класса

#### Класс UserData
Класс отвечает за хранения и логику работы с данными текущего пользователя.\
Конструктор класса принимает инстант брокера событий.\
В полях класса хранятся следующие данные:
- payment: string - способ оплаты
- email: string - почта
- phone: string - телефон 
- address: string - адрес доставки
- total: number - количество товаров в корзине
- items: IItem[] - список товаров в корзине
- events: IEvents - экземпляр класса `EventEmitter` для инициации событий при изменении данных

Класс имеет набор методов для взаимодействия с данными:
- getUserInfo(): TUserPublic - возвращает основные данные пользователя в классе
- setUserInfo(userData: IUser): void - сохраняет данные пользователя в классе

### Классы представления
Все классы представления отвечают за отображение внутри контейнера (DOM-элемент) передаваемых в них данных.

#### Класс Modal
Реализует модальное окно. Так же предоставляет методы `open` и `close` для управления отображением модального окна. Устанавливает слушатели на клавиатуру для закрытия модального окна по Esc, на клик в оверлей и кнопку-крестик для закрытия попапа. Конструктор принимает селектор, по которому в разметке страницы будет идентифицировано модальное окно, DOM-элемент темплейта, которым будет заполняться модальное окно и экземпляр класса `EventEmitter` для возможности инициации событий.

Поля класса:
- modal: HTMLElement - элемент модального окна
- events: IEvents - брокер событий

Методы:
- render(template: HTMLElement): HTMLElement - возвращает модальное окно, заполненное переданным шаблоном

#### Абстрактный класс Form
Отвечает за отображение полей ввода в модальных окнах.

Методы:
- setValid(isValid: boolean): void - изменяет активность кнопки подтверждения
- getInputValues(): Record<string, string> - возвращает объект с данными из полей формы, где ключ - name инпута, значение - данные, введенные пользователем.
- setInputValues(data: Record<string, string>):  void - принимает объект с данными из полей формы.
- get form: HTMLElement - геттер для получения элемента формы

#### Класс FormAddress
Дочерний класс Form. Соответствует форме для заполнения адреса доставки.

#### Класс FormContacts
Дочерний класс Form. Соответствует форме для заполнения контактных данных.

#### Класс MainPage
Отвечает за отображение каталога товаров на главной странице и за отображение иконки корзины с счетчиком количества товаров в ней.\
В конструктор принимает контейнер, в котором размещаются товары, элемент разметки корзины и экземпляр `EventEmitter` для инициации событий.\
В полях класса содержится ссылка на элемент разметки корзины, а так же элемент разметки каталога товаров.\
Методы:
- setUserInfo(userData: TUserPublic): void - устанавливает данные пользователя в элемент разметки корзины

#### Класс Item
Отвечает за отображение карточки товара, задавая данные (категорию, название, изображение, цену). В конструктор передается DOM-элемент темплейта, чтобы формировать товары разных вариантов верстки. В классе устанавливается слушатель на всю карточку товара.\
Поля класса содержат элементы разметки элементов карточки товара. Конструктор, кроме темплейта принимает экземпляр `EventEmitter` для инициации событий.\
Методы:
- setData(itemData: IItem): void - заполняет атрибуты элементов карточки товара данными.
- render(): HTMLElement - метод возвращает полностью заполненную карточку товара с утановленным слушателем
- геттер id возвращает уникальный id товара

#### Класс Basket
Отвечает за отображение списка товаров, выбранных пользователем.
В конструктор принимает список товаров, которые выбрал пользователь, шаблон (template) и экземпляр `EventEmitter` для инициации событий.\
Поля класса включают информацию о товарах, выбранных пользователем (название, цена), а так же порядковый номер.\

Методы:
- deleteItem(itemData: IItem):void - удаляет товар из корзины
- orderItems():void - метод оформления товаров для покупки

#### Класс Address
Отвечает за отображение контента для способа оплаты и адреса доставки.
В конструктор принимает шаблон(template) и экземпляр `EventEmitter` для инициации событий.\
Поля класса содержат элементы разметки элементов модального окна адреса.

#### Класс Contacts
Отвечает за отображение контента для модального окна вводы контактных данных.
В конструктор принимает шаблон(template) и экземпляр `EventEmitter` для инициации событий.\
Поля класса содержат элементы разметки элементов модального окна контактных данных.

#### Класс Success
Отвечает за отображение сообщения об успешном сохранении заказа на сервере.
В конструктор принимает шаблон(template) и экземпляр `EventEmitter` для инициации событий.\
Поля класса содержат изображение, подтверждающую надпись, общую стоимость товаров в корзине и кнопку подтверждения.

### Слой коммуникации

#### Класс AppApi
Принимает в конструктор экземпляр класса Api и предоставляет методы, реализующие взаимодействие с бэкендом сервиса.

## Взаимодействие компонентов
Код, описывающий взаимодействие представления и данных между собой находится в файле `index.ts`, выполняющем роль презентера.\
Взаимодействие осуществляется за счет событий, генерируемых с помощью брокера событий и обработчиков этих событий, описанных в `index.ts`.\
В `index.ts` сначала создаются экземпляры всех необходимых классов, а затем настраивается обработка событий.

Список всех событий, которые могут генерироваться в системе:\
*События изменения данных:*
- `userData:changed` - изменение данных пользователя
- `itemData:selected` - изменение открываемой в модальном окне карточки товара

*События, возникающие при взаимодействии пользователя с интерфейсом:*
- `modal:open` - открытие модального окна
- `modal:input` - ввод данных в поля ввода в модальных окнах
- `modal:submit` - сохранение данных в модальных окнах
- `item:select` - выбор товара для отображения в модальном окне
- `item:put` - добавление товара в корзину
- `item:remove` - удаление товара из корзины
